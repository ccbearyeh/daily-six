3. 為什麼 Side Panel 不自動彈出？

Chrome 限制 chrome.sidePanel.open() 必須在「使用者手勢」事件中呼叫 （點擊工具列按鈕、點擊通知等），不能由 alarm / timer 自動觸發。 這是為了避免擴充功能無端霸佔螢幕。 因此 Daily Six 採「Badge + 通知」的雙軌策略：Badge 提供安靜的視覺指標， 通知用於需要使用者主動接管的情境，並在通知點擊時順勢開啟 Side Panel。

實作要點

  • Badge 更新邏輯封裝在 background.ts，避免散佈在 React view 中。
  • 所有通知標題／內文走 i18n key（見 SPEC §8.3），讓繁中 / 英文版本一致。
  • 停滯提醒需偵測「工作時段」設定（預設 09:00–18:00），避免在深夜送出。
  • 通知去重：同一時段不重複發送（用 chrome.notifications.create(notificationId) 的相同 ID 覆蓋）。
  • OS 通知被封鎖時，App 內顯示 banner 提示使用者前往系統設定授權。
